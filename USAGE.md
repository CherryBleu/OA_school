# 校园小组协同 OA 系统使用说明

## 1. 系统角色

系统分为两类用户：

- 组长：创建项目、小组号分享、需求拆解、任务派发、查看团队数据大屏。
- 组员：维护技能画像、完成 AI 技能核验、加入项目组、处理任务、提交 Git 仓库链接。

组长同时也是项目开发成员，也可以参与技能核验、领取任务和提交代码。

## 2. 登录与注册

1. 打开前端系统地址。
2. 新用户点击“注册”，填写学号/账号、姓名和密码。
3. 已有用户点击“登录”，输入账号和密码。
4. 登录成功后进入协同工作台。

系统会自动在浏览器中生成 `device_uuid`，用于识别当前设备和支持多端在线管理。

## 3. 技能画像与 AI 核验

1. 进入左侧“技能”页面。
2. 输入技能标签，例如 `Java`、`Vue`、`PostgreSQL`、`UI`。
3. 选择自评星级，范围为 1 到 5 星。
4. 点击“保存”。
5. 点击该技能后的“核验”，系统会生成 AI 试卷。
6. 完成答题后点击“提交阅卷”。
7. AI 返回 0 到 100 分，分数大于等于 60 分时，该技能会被标记为“金标”。

注意：用户至少需要通过一个 AI 技能核验，才能加入项目组。

## 4. 创建项目和小组号

组长使用：

1. 进入左侧“项目”页面。
2. 在“创建项目”区域填写项目名称和项目说明。
3. 点击“创建”。
4. 系统会自动生成一个业务小组号，例如 `A7K29QX`。
5. 组长把该小组号分享给组员。

小组号用于入组，不暴露数据库主键。

## 5. 加入项目组

组员使用：

1. 进入左侧“项目”页面。
2. 在“加入小组”区域输入组长提供的小组号。
3. 输入框失焦或点击“预检”后，系统会显示项目名、组长姓名和当前人数。
4. 确认无误后点击“确认加入”。
5. 如果还没有通过 AI 技能核验，系统会拒绝入组，需要先去“技能”页面完成核验。

## 6. AI WBS 任务拆解

组长使用：

1. 进入项目工作台。
2. 点击“WBS 拆解”标签页。
3. 粘贴课程作业、竞赛题目或项目需求文本。
4. 点击“AI 拆解任务”。
5. 系统会自动生成任务卡片，每个任务包含：
   - 任务标题
   - 任务说明
   - 任务类别
   - 预计工期
   - 硬性功能清单 `features_json`
   - 推荐负责人

`features_json` 是后续 AI 源码审查的基线。

## 7. 敏捷看板使用

进入项目工作台后，默认显示“敏捷看板”。

任务状态包括：

- 待开始
- 进行中
- 审查中
- 已完成

用户可以在任务卡片中：

- 修改负责人
- 修改任务状态
- 查看硬性功能清单
- 查看 AI 审查结果
- 提交 GitHub/Gitee 仓库链接

## 8. 提交源码与 AI 审查

1. 在任务卡片底部输入 GitHub 或 Gitee 仓库地址。
2. 点击“提交”。
3. 后端会拉取远程仓库源码。
4. 系统会过滤 `.git`、`node_modules`、`target`、`dist` 等非核心目录。
5. AI 会对照该任务的 `features_json` 逐项审查。
6. 如果全部通过，任务状态变为“已完成”。
7. 如果存在未实现功能，任务会回到“进行中”，并显示 AI 打回评语。

只支持 GitHub/Gitee 仓库链接，不支持上传本地压缩包。

## 9. 数据大屏

进入项目工作台后点击“数据大屏”。

系统展示：

- 总进度环形图
- WBS 燃尽图
- 成员贡献率饼图

项目进度不是简单按任务数量计算，而是按任务预计工期加权计算。已完成任务按 100% 计入，被 AI 打回的任务按残余完成度计入。

## 10. 在线设备管理

1. 点击左侧“设备”页面。
2. 查看当前账号已登录的设备列表。
3. 当前设备会显示“当前”标签。
4. 对其他设备点击“下线”，系统会使该设备登录失效。

如果被踢下线的设备正在打开系统，前端会收到 `FORCE_LOGOUT` 消息并回到登录页。

## 11. 常见问题

### 后端启动时报 `Connection to localhost:5432 refused`

说明 PostgreSQL 没有启动，或 `backend/.env` 中数据库地址不正确。

可以使用 Docker：

```bash
npm run db:up
```

也可以安装 Windows 版 PostgreSQL，并确认：

```bash
DATABASE_URL=jdbc:postgresql://localhost:5432/oa_school
DB_USERNAME=postgres
DB_PASSWORD=你的PostgreSQL密码
```

### Docker Desktop 提示 `Virtualization support not detected`

说明 Windows 没有开启虚拟化。可以进入 BIOS/UEFI 开启 Intel VT-x 或 AMD-V，或者直接安装 Windows 版 PostgreSQL，不使用 Docker。

### AI 功能没有真实调用大模型

检查后端环境变量：

```bash
LLM_API_KEY=你的 key
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
```

如果没有配置 `LLM_API_KEY`，系统会使用本地降级逻辑，方便演示流程。

## 12. 推荐演示流程

1. 注册组长账号。
2. 在“技能”页面添加并核验一个技能。
3. 创建项目，复制小组号。
4. 注册组员账号。
5. 组员添加并核验技能。
6. 组员用小组号预检并加入项目。
7. 组长进入项目，粘贴需求并执行 AI WBS 拆解。
8. 在看板中查看任务推荐负责人。
9. 组员提交 GitHub/Gitee 仓库链接。
10. 查看 AI 审查结果和数据大屏变化。
